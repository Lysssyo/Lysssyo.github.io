# 图书馆预约系统设计（自托管）

## 背景

原方案（见 [010-图书馆预约系统设计（FC + GitHub Actions）](010-图书馆预约系统设计（FC%20+%20GitHub%20Actions）.md)）链路较长：

```
阿里云 FC 定时器 → FC 触发函数 → GitHub Actions → Playwright 预约
```

存在的问题：
- GitHub Actions 延迟大，每次启动都要重新下载依赖
- 阿里云 FC 增加了一层间接调用和运维成本
- 修改 `seat_id` 需要通过 GitHub API 操作 Actions Variables

迁移到自有服务器后，链路简化为：

```
systemd timer → run_once.py → Playwright 预约
```

代码仓库：`Lysssyo/library-reservation(self-hosted)`

---

## 整体架构

```
systemd timer（定时触发）
    ↓
run_once.py（任务入口）
    ├── 获取文件锁（防并发）
    ├── 读取 state.json（enabled、seat_id）
    ├── 调用 run_reserve(seat_id)
    ├── 写入 logs/YYYY-MM-DD.log
    └── 更新 state.json（last_run_at、last_result）

前端管理页面
    ↓ HTTPS
Nginx 反向代理（443 → 127.0.0.1:8000）
    ↓
server.py（Flask API）
    ├── GET  /status     查看状态
    ├── POST /enabled    开关任务
    ├── GET  /seat       查看座位
    ├── POST /seat       修改座位
    ├── POST /run        手动触发
    └── GET  /logs       查看日志
```

两条线共享 `state.json`、`logs/`、`runtime/reserve.lock`。

## 与旧方案的对比

| 维度   | FC + GitHub Actions                 | 自托管                       |
| ---- | ----------------------------------- | ------------------------- |
| 定时触发 | FC 定时器 → FC 函数 → GitHub dispatches  | systemd timer 直接调用脚本      |
| 执行环境 | GitHub Actions runner（每次装依赖）        | 本地 venv（依赖预装）             |
| 配置管理 | GitHub Actions Variables / Secrets  | state.json + /etc/app.env |
| 控制接口 | FC HTTP 函数 → FC 管理 API + GitHub API | Flask 直接读写 state.json     |
| 延迟   | FC 触发 + Actions 排队，可能延迟数分钟          | systemd 直接启动，秒级           |

## 核心设计

### 控制面与执行面分离

```
server.py   → 控制面（常驻，轻量，只读写 state.json 和日志）
run_once.py → 执行面（按需启动，跑完退出，释放内存）
```

Playwright / Chromium 只在 `run_once.py` 执行期间启动，任务结束后进程退出，不常驻。API 服务只做 JSON 读写，内存占用很小。

### 文件锁防并发

`run_once.py` 启动时对 `runtime/reserve.lock` 加排他锁（`fcntl.flock`）。如果已有任务在运行，新启动的实例会立即退出，避免多个 Chromium 进程同时运行。

### Tee 双写日志

`run_once.py` 通过替换 `sys.stdout` / `sys.stderr` 为 `Tee` 对象，将所有 `print()` 输出同时写入日志文件和终端。业务脚本（`refresh_all.py`、`utils.py`）无需任何修改即可被记录。

### 业务开关

`POST /enabled` 修改 `state.json` 中的 `enabled` 字段。systemd timer 仍然按时触发，但 `run_once.py` 发现 `enabled=false` 后直接退出。好处是不需要操作 `systemctl`，权限简单，日志中也能看到"任务关闭，已跳过"。

## 定时策略

systemd timer 配置（Asia/Shanghai 时区）：

```ini
OnCalendar=*-*-* 07:00:00 Asia/Shanghai
OnCalendar=*-*-* 08..21:00:00 Asia/Shanghai
OnCalendar=*-*-* 08..21:25:00 Asia/Shanghai
OnCalendar=*-*-* 08..21:50:00 Asia/Shanghai
```

即北京时间 07:00 一次，08:00-21:00 每 25 分钟一次（:00、:25、:50）。

## 网络架构

```
浏览器 → https://lib.zl6544hx.asia
         ↓
Nginx（监听 443，SSL 证书 by Let's Encrypt）
         ↓ proxy_pass
Flask（127.0.0.1:8000，不对外暴露）
```

> [!TIP]
> 浏览器看到 https:// 协议前缀时，默认连接目标服务器的 443 端口。

SSL 证书由 Certbot 自动续期。
