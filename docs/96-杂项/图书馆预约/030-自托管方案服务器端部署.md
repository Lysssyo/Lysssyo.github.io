# 自托管方案服务器端部署

## 环境信息

- 服务器：阿里云，IP `8.166.142.73`
- 部署路径：`/opt/gzhu-lib-reserve`
- Python 虚拟环境：`/opt/gzhu-lib-reserve/venv`
- 敏感配置：`/etc/gzhu-lib-reserve/app.env`
- systemd 配置：`/etc/systemd/system/`
- 域名：`lib.zl6544hx.asia`

---

## 1. 拉取代码

```bash
git clone -b self-hosted https://github.com/Lysssyo/library-reservation.git /opt/gzhu-lib-reserve
cd /opt/gzhu-lib-reserve
```

## 2. 创建虚拟环境并安装依赖

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
playwright install chromium
```

## 3. 创建环境变量文件

```bash
mkdir -p /etc/gzhu-lib-reserve
cat > /etc/gzhu-lib-reserve/app.env << 'EOF'
LIB_USER=你的学号
LIB_PASS=你的密码
EOF
chmod 600 /etc/gzhu-lib-reserve/app.env
```

## 4. 初始化 state.json

```bash
cat > /opt/gzhu-lib-reserve/state.json << 'EOF'
{
  "enabled": true,
  "seat_id": "101267824",
  "last_run_at": null,
  "last_result": null
}
EOF
```

## 5. 部署 systemd 配置

```bash
cp /opt/gzhu-lib-reserve/deploy/*.service /etc/systemd/system/
cp /opt/gzhu-lib-reserve/deploy/*.timer /etc/systemd/system/
systemctl daemon-reload
```

启动 API 服务和定时任务：

```bash
systemctl enable --now gzhu-lib-reserve-api.service
systemctl enable --now gzhu-lib-reserve-task.timer
```

## 6. Nginx 反向代理 + HTTPS

### 6.1 安装 Nginx 和 Certbot

```bash
apt update && apt install -y nginx
apt install -y certbot python3-certbot-nginx
```

### 6.2 创建 Nginx 配置

配置文件写入 `/etc/nginx/sites-available/`，这是 Debian/Ubuntu 打包 Nginx 时约定的目录结构：

- `sites-available/`：存放所有站点配置文件（不管是否启用）
- `sites-enabled/`：存放软链接，指向 `sites-available/` 里要启用的配置
- Nginx 主配置 `/etc/nginx/nginx.conf` 中有 `include /etc/nginx/sites-enabled/*;`，只有 `sites-enabled/` 里的配置才会生效

先写一个只有 HTTP 的配置，确认反向代理能通：

```bash
cat > /etc/nginx/sites-available/gzhu-lib-reserve << 'EOF'
server {
    listen 80;
    server_name lib.zl6544hx.asia;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
EOF

ln -s /etc/nginx/sites-available/gzhu-lib-reserve /etc/nginx/sites-enabled/
nginx -t && systemctl reload nginx
```

验证：`curl http://lib.zl6544hx.asia/status` 应返回 state.json 内容。

### 6.3 申请 SSL 证书

前提：域名 `lib.zl6544hx.asia` 的 A 记录已指向服务器 IP。

```bash
certbot certonly --nginx -d lib.zl6544hx.asia --agree-tos -m 你的邮箱
```

这条命令做两件事：

1. **域名验证**：`--nginx` 让 Certbot 借用 Nginx 完成验证。具体过程是：Certbot 找到 `server_name` 与 `-d` 参数匹配的 server 块，临时注入一条 `location /.well-known/acme-challenge/` 规则并 reload Nginx。Let's Encrypt 服务器通过 HTTP 访问该路径读取验证令牌，确认域名所有权后，Certbot 自动删除临时规则，恢复原配置
2. **保存证书**：`certonly` 表示只申请证书不永久修改 Nginx 配置，证书保存到固定路径：
   - 证书：`/etc/letsencrypt/live/lib.zl6544hx.asia/fullchain.pem`
   - 私钥：`/etc/letsencrypt/live/lib.zl6544hx.asia/privkey.pem`

证书有效期 90 天。Certbot 安装时自动创建了 `certbot.timer`（`systemctl status certbot.timer` 确认），每天检查两次，到期前 30 天内自动续期。

### 6.4 更新 Nginx 配置为 HTTPS

证书申请成功后，手动将 Nginx 配置更新为 HTTPS 版本：

```bash
cat > /etc/nginx/sites-available/gzhu-lib-reserve << 'EOF'
server {
    listen 80;
    server_name lib.zl6544hx.asia;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl;
    server_name lib.zl6544hx.asia;

    ssl_certificate /etc/letsencrypt/live/lib.zl6544hx.asia/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/lib.zl6544hx.asia/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
EOF

nginx -t && systemctl reload nginx
```

配置说明：

- 第一个 server 块：80 端口接收 HTTP 请求，301 重定向到 HTTPS
- 第二个 server 块：443 端口处理 HTTPS，`ssl_certificate` 和 `ssl_certificate_key` 指向 Certbot 保存证书的路径，`proxy_pass` 将请求转发到 Flask（127.0.0.1:8000）

### 6.5 阿里云安全组

需要开放以下入站端口：
- 80（HTTP，用于重定向到 HTTPS）
- 443（HTTPS）

## 日常运维

### 代码更新

```bash
cd /opt/gzhu-lib-reserve
git pull
systemctl restart gzhu-lib-reserve-api.service
```

如果 timer 配置也有变更：

```bash
cp deploy/*.timer /etc/systemd/system/
systemctl daemon-reload
systemctl restart gzhu-lib-reserve-task.timer
```

### 查看服务状态

```bash
systemctl status gzhu-lib-reserve-api.service
systemctl status gzhu-lib-reserve-task.timer
systemctl list-timers gzhu-lib-reserve*
```

### 查看日志

```bash
# 业务日志（按天切分）
cat /opt/gzhu-lib-reserve/logs/$(date +%Y-%m-%d).log

# systemd 服务日志
journalctl -u gzhu-lib-reserve-api.service --no-pager -n 50
journalctl -u gzhu-lib-reserve-task.service --no-pager -n 50
```
